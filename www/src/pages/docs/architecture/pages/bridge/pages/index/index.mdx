# Bridge Yapıları

Daemon, sistem düzeyindeki işlemleri native bridge yapıları aracılığıyla gerçekleştirir. Python katmanı iş mantığını yönetir, bridge yapıları ise kernel ile doğrudan iletişim kurar. Her iki bridge `ctypes` FFI üzerinden `.so` shared library olarak yüklenir.

<Mermaid chart={`flowchart TD
  D["Phantom Daemon\\n(Python)"]
  D -->|"ctypes FFI"| WGB["wireguard-go-bridge\\n(Go · .so)"]
  D -->|"ctypes FFI"| FWB["firewall-bridge\\n(Rust · .so)"]
  WGB -->|"tun.CreateTUN"| TUN["/dev/net/tun\\nWireGuard arayüzleri"]
  FWB -->|"nftables API"| NFT["nftables\\nKernel kuralları"]
`} />

| Bridge                  | Dil  | Sorumluluk                                                               |
|-------------------------|------|--------------------------------------------------------------------------|
| **wireguard-go-bridge** | Go   | WireGuard userspace, TUN arayüz oluşturma, peer yönetimi, UAPI protokolü |
| **firewall-bridge**     | Rust | nftables kural grupları, routing policy, atomik kural uygulaması         |

---

## wireguard-go-bridge

WireGuard userspace implementasyonunu sağlar. Her arayüz için ayrı bir bridge instance oluşturulur.

### Yükleme

```python
from wireguard_go_bridge import WireGuardBridge

bridge = WireGuardBridge(ifname="wg_phantom_main", mtu=1420, db_path="/var/lib/phantom/state/db/wireguard/wg_phantom_main/device.db")
```

### Yaşam Döngüsü

<Mermaid chart={`flowchart
  A["WireGuardBridge()"] --> B["up()"]
  B --> C["ipc_set / ipc_get"]
  C --> D["down()"]
  D --> E["close()"]
`} />

### IPC Konfigürasyon Formatı (UAPI)

```text
private_key={hex}
listen_port={int}
replace_peers=true
public_key={hex}
preshared_key={hex}
allowed_ip={ipv4}/32
allowed_ip={ipv6}/128
persistent_keepalive_interval={int}
```

### Anahtar Üretimi

```python
from wireguard_go_bridge.keys import generate_private_key, derive_public_key, generate_preshared_key

private = generate_private_key()       # Curve25519 (hex)
public  = derive_public_key(private)   # Public key (hex)
psk     = generate_preshared_key()     # PSK (hex)
```

### Hata Türleri

| Exception           | Açıklama                         |
|---------------------|----------------------------------|
| `BridgeError`       | Temel hata sınıfı                |
| `TunCreateError`    | TUN arayüz oluşturma hatası      |
| `DeviceCreateError` | Cihaz oluşturma hatası           |
| `IpcError`          | IPC iletişim hatası              |
| `DeviceUpError`     | Arayüz etkinleştirme hatası      |
| `DeviceDownError`   | Arayüz devre dışı bırakma hatası |

## firewall-bridge

nftables kural yönetimini sağlar. Grup tabanlı kural organizasyonu, atomik uygulama ve preset sistemi destekler.

### Yükleme

```python
from firewall_bridge import FirewallBridge

bridge = FirewallBridge(db_path="/var/lib/phantom/state/db/firewall.db")
```

### Preset Sistemi

Preset'ler YAML formatında tanımlanır ve değişken interpolasyonu destekler:

#### core.yaml

```yaml
name: core
priority: 50
type: system
metadata:
  description: "Core VPN firewall — listen port, forwarding, masquerade"

rules:
  # wg-listen
  - chain: input
    action: accept
    proto: udp
    dport: "{listen_port}"

  # established-in
  - chain: input
    action: accept
    state: "established,related"

  # wg-forward-in
  - chain: forward
    action: accept
    in_iface: "{wg_interface}"

  # wg-forward-out
  - chain: forward
    action: accept
    out_iface: "{wg_interface}"
    state: "established,related"

  # vpn-masquerade
  - chain: postrouting
    action: masquerade
    source: "{ipv4_subnet}"
```

#### multihop-exit.yaml

```yaml
name: multihop-exit
priority: 80
type: system
metadata:
  description: "Multihop exit routing — main VPN to exit tunnel"

table:
  - ensure: {id: 200, name: mh}
  - policy: {from: "{ipv4_subnet}", table: mh, priority: 100}
  - route: {destination: "{ipv4_subnet}", device: "{wg_interface}", table: mh}
  - route: {destination: default, device: "{wg_interface_exit}", table: mh}

rules:
  # main → exit forwarding
  - chain: forward
    action: accept
    in_iface: "{wg_interface}"
    out_iface: "{wg_interface_exit}"

  # exit → main return traffic
  - chain: forward
    action: accept
    in_iface: "{wg_interface_exit}"
    out_iface: "{wg_interface}"
    state: "established,related"

  # exit masquerade
  - chain: postrouting
    action: masquerade
    source: "{ipv4_subnet}"
    out_iface: "{wg_interface_exit}"
```
