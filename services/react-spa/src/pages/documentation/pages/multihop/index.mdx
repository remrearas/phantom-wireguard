# Multihop

Multihop, istemci trafiğini doğrudan internete çıkarmak yerine bir **exit sunucusu** üzerinden yönlendirir. Daemon iki WireGuard arayüzü çalıştırır: biri istemcilere hizmet verir (`wg_phantom_main`), diğeri exit tüneline bağlanır (`wg_phantom_exit`). Aradaki trafik aktarımını firewall kuralları sağlar.

## Trafik Akışı

İstemciden gelen bir paket daemon'a ulaştığında firewall kuralları devreye girer:

<Mermaid chart={`sequenceDiagram
  participant C as İstemci
  participant M as wg_phantom_main
  participant FW as Firewall (nftables)
  participant E as wg_phantom_exit
  participant X as Exit Sunucusu

  C->>M: Şifreli paket (WireGuard)
  M->>FW: Çözülmüş paket
  FW->>FW: forward: main→exit (accept)
  FW->>FW: postrouting: masquerade (SNAT)
  FW->>E: Yönlendirilmiş paket
  E->>X: Şifreli paket (WireGuard tünel)
  X-->>E: Yanıt
  E-->>FW: Çözülmüş yanıt
  FW-->>FW: forward: exit→main (established/related)
  FW-->>M: Yanıt paketi
  M-->>C: Şifreli yanıt
`} />

## Firewall Kuralları

Multihop etkinleştirildiğinde `multihop-exit` preset'i uygulanır. Üç temel kural grubu vardır:

### Yönlendirme (Policy Routing)

Tablo 200 (`mh`) oluşturulur. İstemci alt ağından gelen trafik bu tabloya yönlendirilir:

<Mermaid chart={`flowchart TD
  A["Kaynak: istemci alt ağı\n(ör. 10.8.0.0/24)"] -->|"policy rule"| T["Tablo 200 (mh)"]
  T --> R1["10.8.0.0/24 → wg_phantom_main\n(yerel VPN trafiği)"]
  T --> R2["default → wg_phantom_exit\n(diğer tüm trafik)"]
`} />

### Forward Kuralları

```text
main → exit :  accept                       (tüm istemci trafiği geçer)
exit → main :  accept (established/related)  (sadece yanıt trafiği döner)
```

### Masquerade

```text
postrouting :  kaynak=istemci alt ağı, çıkış=wg_phantom_exit → masquerade
```

İstemci IP'leri exit sunucusuna daemon'ın tünel adresi olarak görünür (SNAT).
