## Nasıl Çalışır?

***Terazi***, her iki protokol ailesini tek bir koordinat sistemi üzerinden yönetir: **index**.

Alt ağ oluşturulduğunda IPv4 ve IPv6 adresleri bu indeks üzerinden birbirine kilitlenir:

```text
index 0   →  10.8.0.0        ↔  fd00:70:68::0    (ağ adresi, rezerve)
index 1   →  10.8.0.1        ↔  fd00:70:68::1    (sunucu / gateway, rezerve)
index 2   →  10.8.0.2        ↔  fd00:70:68::2    (1. istemci)
index 3   →  10.8.0.3        ↔  fd00:70:68::3    (2. istemci)
index 4   →  10.8.0.4        ↔  fd00:70:68::4    (3. istemci)
  ...
index n   →  10.8.0.n        ↔  fd00:70:68::n    (n-1. istemci)
  ...
index n-1 →  10.8.0.255      ↔  fd00:70:68::ff   (broadcast, rezerve)
```

Yapının özü şudur: **IPv4 kapasitesi baz alınır, IPv6 kendini buna göre şekillendirir.** Prefix uzunluğu değiştiğinde her iki protokol aynı anda yeniden boyutlanır ve simetri her koşulda korunur.

## Prefix Dönüşümü

Referans implementasyon — `phantom_daemon/base/wallet/wallet.py`:

```python
def _calculate_terazi(ipv4_subnet: str) -> tuple[str, str, int, int]:
    """Calculate matching IPv6 subnet from IPv4 subnet.

    Returns (ipv4_str, ipv6_str, capacity, host_bits).
    capacity = usable client slots (total - network - broadcast - server).
    """
    v4_net = ipaddress.ip_network(ipv4_subnet, strict=False)
    host_bits = 32 - v4_net.prefixlen
    v6_prefix = 128 - host_bits
    v6_net = ipaddress.ip_network(f"fd00:70:68::/{v6_prefix}")
    capacity = v4_net.num_addresses - 3  # network + broadcast + server
    return str(v4_net), str(v6_net), capacity, host_bits
```

Her iki protokol aynı host bit sayısını paylaşır. IPv4 prefix'i, IPv6 prefix'ini doğrudan belirler — iki satırlık bir dönüşüm, iki havuzun simetrisini garanti eder:

  A["/24 — IPv4"] -->|"32 − 24 = 8 host bit"| B["/120 — IPv6"]
  B -->|"2⁸ − 3 = 253 slot"| C["253 istemci kapasitesi"]
`} />

## Havuz Yapısı

Adresler veritabanında önceden oluşturulur (`wallet.db` → `users` tablosu). Her satır bir IPv4/IPv6 çiftidir.
Boş slot'lar `<null>` olarak bekler. İstemci atandığında anahtar, isim ve zaman damgası doldurulur.

## CIDR Değişikliğinde Ne Olur?

## Doğrulama

`GET /api/core/network/validate` uç noktası havuzun terazi simetrisini ve yapısal bütünlüğünü doğrular. Başarılı yanıt:

```json
{
  "ok": true,
  "data": {
    "valid": true,
    "errors": []
  }
}
```

Terazi index uyumsuzluğu veya adres boşluğu tespit edilirse `valid: false` ve hata listesi döner:

```json
{
  "ok": true,
  "data": {
    "valid": false,
    "errors": [
      "Row 50: terazi mismatch v4_index=50 v6_index=51",
      "Row 100: IPv4 gap 10.8.0.100 -> 10.8.0.102"
    ]
  }
}
```
